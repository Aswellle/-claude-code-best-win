## 一、需求描述

双击 `packaging/output/ClaudeCode.exe` 后，终端弹出窗口中出现以下错误：

```
'D:\Git' 不是内部或外部命令，也不是可运行的程序或批处理文件。
C:\Users\welle>
```

而非正常进入 Claude Code Best 的 TUI 工作流。

---

## 二、根因分析

`launcher.ps1` 中通过以下方式启动终端：

```powershell
$WtArgs = "--title `"$WindowTitle`" cmd /k `"$CoreExe`" $ArgString"
Start-Process -FilePath $WtExe -ArgumentList $WtArgs
```

生成的命令行为：

```
wt.exe --title "Claude Code Best" cmd /k "D:\Git Clone\CC-SRC\V5\packaging\output\claude-core.exe" --cwd ...
```

**问题**：`cmd.exe` 处理 `/k "..."` 时会剥离外层引号，剥离后得到：

```
D:\Git Clone\CC-SRC\V5\packaging\output\claude-core.exe
```

cmd.exe 在没有引号保护的情况下按空格分割，将 `D:\Git` 视为命令名，剩余部分作为参数，导致"不是内部命令"报错。

同样问题存在于无 wt.exe 时的 cmd.exe 直接回退分支。

---

## 三、解决方案

使用 **双层双引号（double-double-quote）** 模式：`cmd /k ""exe with spaces"" args`

- 外层 `"..."` 是 cmd.exe `/k` 参数的包裹引号，cmd.exe 剥离它
- 剥离后剩余：`"D:\Git Clone\...\claude-core.exe" args`
- 内层引号正确保护了含空格的路径，cmd.exe 能正确解析可执行文件

修改后两个分支：

```powershell
# wt.exe 分支
$WtArgs = "--title `"$WindowTitle`" cmd /k `"`"$CoreExe`" $ArgString`""

# cmd.exe 回退分支
Start-Process -FilePath $CmdExe -ArgumentList "/k `"`"$CoreExe`" $ArgString`""
```

## 六、开发总结

根因是 `cmd.exe /k "quoted path"` 的经典陷阱：外层引号被剥除后路径按空格分割。  
修复使用标准 double-double-quote 模式，launcher 和安装包已全部重建。  
本次无需重编译 `claude-core.exe`（`-SkipBuild`），只重编译了 37KB 的 ps2exe launcher。
